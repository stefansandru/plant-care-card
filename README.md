# Plant Care RAG API: Plant Identification and Care

## What problem does it solve?
Often, people have houseplants or garden plants for which they do not know the proper care, or they even forget their names. 
This application solves the problem very simply: **you take a photo of the plant**, and the system automatically identifies it and generates a complete care card, providing clear instructions about light, water, or humidity requirements, as well as details about toxicity or common diseases.

## Architecture and Flow (Multi-Modal Agricultural Assistants)

The developed application is a modern API that combines the flexibility of neural networks with applied RAG intelligence. The application flow consists of 2 major interconnected steps:

1. **Visual Classification Module:** The API endpoint receives an image. This passes through the **PyTorch EfficientNet-B1** model, which quickly identifies and "labels" the plant and calculates the confidence percentage.
2. **Care & RAG Module (LangGraph Pipeline):** The predicted plant name automatically triggers a 3-node graph-directed flow (`LangGraph`):
   - *Researcher Agent:* Searches the web in real-time (via `Tavily API`) for detailed requirements about that specific plant.
   - *Generator Agent:* Analyzes the search results via LLM (`Mistral AI`) and precisely constructs a care dictionary dictated by a strict `Pydantic` model (`PlantCareCard`). 
   - *Validator Agent:* The AI self-verifies; if the generated card has critical missing information, it commands the rewriting of data, otherwise it is returned to the client.

### Motivation - Why this solution?
* **Zero descriptive effort:** The user does not need to describe the plant, a photo is enough. The CNN network prevents textual typing errors.
* **Dynamic Technology:** RAG generates these documents *on-the-fly* with real web expertise, eliminating the need for a massive static database, which would be impossible to maintain for tens of thousands of plants.
* **Production Efficiency:** `EfficientNet-B1` is ultra-fast and lightweight.

## API Request (Input)
The `/api/v1/plant-care` endpoint expects a `POST` request containing an image sent as `multipart/form-data`. (Supports formats: `.jpg`, `.png`, `.webp`, `.bmp`, `.tiff`)

## API Endpoints
- `POST /api/v1/plant-care` — image → classification + PlantCareCard (requires RAG env vars)
- `POST /api/v1/predict` — image → classification + ranked label list

## Configuration (RAG)
The RAG pipeline requires environment variables:
- `MISTRAL_API_KEY`
- `TAVILY_API_KEY`
Optional:
- `LLM_MODEL`

## API Response (Output)
The `/api/v1/plant-care` endpoint returns a complex **JSON** that contains both the image model prediction and the detailed care card (generated by the LLM). Output example:
```json
{
  "error": false,
  "classification": {
    "label": "soybeans",
    "confidence": 0.87
  },
  "plant_care_card": {
    "common_name": "Soybean",
    "scientific_name": "Glycine max",
    "care_requirements": {
      "light": { "level": "BRIGHT_DIRECT" },
      "water": { "frequency": "MODERATE" },
      "...": "..."
    },
    "...": "..."
  }
}
```

The `/api/v1/predict` endpoint returns top-1 plus the full ranked list of labels and confidences:
```json
{
  "error": false,
  "results": {
    "label": "soybeans",
    "confidence": 0.87,
    "top_labels": ["soybeans", "orange", "ginger"],
    "top_confidences": [0.87, 0.08, 0.05]
  }
}
```

## Why EfficientNet-B1 (CNN)?
The application uses the pre-trained **EfficientNet-B1** model for initial recognition. The decision is based on an optimal compromise between memory consumption and accuracy obtained compared to larger variants (ResNet18 or B3):

| Model | Parameter Count (Millions) | Accuracy |
| :--- | :---: | :---: |
| EfficientNet B0 | 4.0 | 85.6% |
| **EfficientNet B1** | **6.6** | **90.2%** |
| EfficientNet B2 | 7.7 | 91.7% |
| ResNet18 | 11.2 | 84.2% |

Although B2 has a marginally better accuracy by 1.5%, we chose B1 because it uses approximately 1.1M fewer parameters, facilitating the generation of extremely fast predictions.

## Experimental Results (PyTorch Model)

- **Dataset**: Real data from Kaggle: [Plants Type Datasets](https://www.kaggle.com/datasets/yudhaislamisulistya/plants-type-datasets) (RGB images of plants, 30 final classes).
- **Tracked Metrics**: Accuracy, Precision (macro), Recall (macro), F1 (macro). Loss function: `CrossEntropyLoss`.
- **Preprocessing**: `32x32` resizing, ImageNet normalization, visual augmentations at training (`RandomHorizontalFlip`, `RandomRotation`).
- **Tested Accuracy**: The model reached `val_acc ≈ 0.873` (at only Epoch 2), demonstrating efficient and fast learning via transfer learning with the EfficientNet-B1 backbone.
